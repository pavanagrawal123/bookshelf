Code Overview

The code is broken into different sections for clarity in the code. You can find the models of the data in the models/ folder. The data is defined as a model, as it makes for easier access down the road and much nicer code.

The use of Django as the application framework allows for interesting code, that is much more concise, as it is purpose built for tasks like this. If you are used to Ruby code, Django is very similar to Ruby on Rails, another very popular web development framework.

All the code is deployed using Docker to now.sh. now.sh Allows the use of serverless docker, which is very interesting. It provides sub 500ms cold start times, along with a globally deployed CDN, which has been very speedy in previous use. It also makes running this site very low cost. It is currently running on the free plan and still running with great speed, security, and ease of use.

In order to reduce the complexity of the frontend, there was no usage of a heavy frontend framework like React or Vue. There is no place for those here, as the functionality we are looking for can be driven via pure JS.

Data Security and Backup

The data is stored in the cloud on AWS through an abstraction from now.sh. AWS is trusted by the CIA, NSA, and many other government organizations. It has been thoroughly vetted in the past in order to verify that it meets security standards. Data is also backed up through AWS every few hours, giving us a safety, if anything were to happen to the application. This is known as dynamic backup.

If Then Behavior

In the book view, if the book is assigned to a student, the button changes to a delete button, rather than assign button, saving time.
